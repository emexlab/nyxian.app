# Contributing to this Website

## Quickstart

You can click the `Edit this page` button at the bottom of any page to start editing it on GitHub or you can go to [emexlab/emexlab.github.io](https://github.com/emexlab/emexlab.github.io/tree/main/).

:::tip[general tip]

You can find more documentation at [docusaurus.io](https://docusaurus.io/docs/category/guides).

:::

## Writing Documentation

All folders must contain an `index.mdx`. These and all `.mdx` files directly in [/docs](https://github.com/emexlab/emexlab.github.io/tree/main/docs/) should also contain a value for `sidebar_position` in the metadata.

``` mdx title="example-doc.mdx"
---
sidebar_position: 2
---

# Example Documentation

...
```

:::note

Please follow the best pratices from [markdownguide.org/basic-syntax](https://www.markdownguide.org/basic-syntax/)

:::

:::tip

There is a [Todo](https://github.com/emexlab/emexlab.github.io/emexDE#Todo) list in the README for the emexDE docs.  
This is a good starting point for first contributors

:::

## React
You can also contribute to the website in various other ways:
- Improving [components](https://github.com/emexlab/emexlab.github.io/tree/main/src/components/)
- Creating new components (your component must be used in atleast on page)
- Helping on [pages](https://github.com/emexlab/emexlab.github.io/emexDE/tree/main/src/pages/), like the index page

But please restraint from using swizzling to change the website (except if it's the offical way). Always look if what you're trying to do can be achieved using classes and CSS.

:::note

When writing react code please follow the best pratices from [react.dev](https://react.dev)

:::

:::warning

Changes to `/static/bootstrap` and any `credits.mdx` will always be rejected!

:::
